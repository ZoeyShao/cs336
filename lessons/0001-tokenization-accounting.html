核心模型

一个最小 language model pipeline 可以写成： raw text -> tokenizer -> token ids -> Transformer -> logits -> cross entropy。 第一课只盯住前两步和最后的计量方式。

对象它回答的问题常见坑
Tokenizer 文本如何变成整数序列？ 把 token 当 word；忽略 Unicode、byte fallback、特殊 token。
Vocabulary 整数 id 的取值范围有多大？ vocab 越大，embedding 和输出层通常越贵。
Context length 一次前向最多看多少 token？ 把字符数、词数、token 数混为一谈。
Loss 模型平均每个预测 token 错多少？ 忘记 loss 的单位通常是 per token。
第一课的金句： tokenization 不是预处理小事，而是模型接口。接口一变，训练样本长度、embedding 参数、loss 可比性、数据预算都会变。

小练习 1：参数量心算

只看 token embedding 和输出层。设词表大小为 V，hidden size 为 d。 如果 embedding 与 LM head 不共享权重，参数大约是 2 * V * d；共享权重时大约是 V * d。
