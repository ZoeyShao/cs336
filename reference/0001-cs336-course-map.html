Stanford CS336: Language Modeling from Scratch

课程地图 | Spring 2026 | 整理日期：2026-06-28

这门课的核心价值不是“又讲一遍 LLM”，而是把语言模型训练栈拆成可以实现、测量和取舍的工程系统：数据进入 tokenizer，token 进入 Transformer，训练跑在 GPU 和分布式系统上，scaling law 指导预算，评测暴露能力边界，post-training 改变模型行为。

一眼看懂

目标 从零开发自己的语言模型：数据收集与清洗、tokenization、Transformer 架构、训练、评测、部署前判断。
难点 作业代码量很大，且系统部分要求理解 GPU memory hierarchy、profiling、Triton kernel 和 distributed training。
适合你怎么学 先用课程补 implementation gap，再用作业暴露自己对 tensor shape、数值稳定性、资源估算和训练数据质量的盲点。
不要怎么学 不建议只刷视频。CS336 的“肉”在作业和实验：每个模块都要能写、能测、能解释失败原因。

课程主线

模块Lectures你应该带走什么
Basics 1-4 Tokenization、resource accounting、Transformer architecture、attention alternatives、MoE。重点是 shape、参数量、FLOPs、训练稳定性。
Systems 5-8 GPU/TPU、kernels、Triton/XLA、parallelism。重点是 memory hierarchy、arithmetic intensity、communication 和 latency hiding。
Scaling 9, 11 用小规模实验拟合和外推。重点是 compute budget 下 model size、data size、loss 的取舍。
Inference & Eval 10, 12 推理解码、serving bottleneck、benchmark 的信号与噪声。重点是不要把 benchmark 当成能力本体。
Data 13-14 Common Crawl、过滤、去重、数据混合、合成数据。重点是数据管线通常比模型结构更决定可用性。
Post-training 15-17 SFT、RLHF/RLVR、GRPO、alignment、多模态。重点是 objective mismatch、reward design 和 evaluation leakage。

作业路线

作业主题建议验收标准
A1 Basics BPE tokenizer、Transformer LM、CE loss、AdamW、training loop。 不看答案实现全部接口；能解释每个 tensor 的 shape；能从 loss/perplexity 判断训练是否异常。
A2 Systems Profiling、FlashAttention2 in Triton、memory-efficient distributed training。 能用 profiler 找 bottleneck；能解释为什么 kernel 快或慢；能区分 compute-bound 和 memory-bound。
A3 Scaling 组件消融、训练 API、scaling law 拟合。 能用小实验外推预算；能说清楚外推假设何时会坏。
A4 Data Common Crawl 处理、语言过滤、去重、数据混合。 能把数据清洗选择和验证 loss/benchmark 变化联系起来。
A5 Alignment SFT、GRPO、math reasoning；optional safety/RLHF supplement。 能解释 SFT、preference optimization、RLVR 的 objective 差异和失败模式。

8 周查漏补缺节奏

  1. 第 1 周：看 L1-L2，做 A1 tokenizer 和 resource accounting。输出一页参数量/FLOPs/显存速查。
  2. 第 2 周：看 L3-L4，做 A1 Transformer/loss/AdamW/training loop。重点 debug shape 和数值稳定性。
  3. 第 3 周：看 L5-L6，做 A2 profiling 和 Triton kernel 最小版本。先追正确性，再追速度。
  4. 第 4 周：看 L7-L8，梳理 DP/TP/PP/FSDP/ZeRO。用小模型画通信图。
  5. 第 5 周：看 L9-L11，做 A3。把 Kaplan 和 Chinchilla 的差异写成自己的决策规则。
  6. 第 6 周：看 L10/L12，做 inference/evaluation 笔记。列出 benchmark leakage、contamination、variance 的检查清单。
  7. 第 7 周：看 L13-L14，做 A4 的 offline subset。比较过滤、去重、混合策略带来的训练信号变化。
  8. 第 8 周：看 L15-L17，做 A5 的核心测试。把 SFT、DPO/RLHF、GRPO 的目标函数和数据需求并排比较。

主动回忆问题

  1. 如果 tokenizer 的 vocab size 翻倍，embedding 参数、softmax 成本和训练数据覆盖会怎么变？
  2. 给定 batch、sequence length、hidden size、layers，哪些量影响激活显存，哪些量影响参数显存？
  3. FlashAttention 为什么不改变 attention 数学结果，却能显著减少显存读写？
  4. Scaling law 外推什么时候最容易失效？数据分布、优化器、模型架构、训练长度分别会怎样破坏假设？
  5. SFT、DPO/RLHF、GRPO 各自需要什么形式的数据？优化信号和风险有什么不同？
