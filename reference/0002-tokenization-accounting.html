Tokenization 与语言模型计量速查

配套 lesson: 第一课：Tokenization 与语言模型计量

词表与 token

Token tokenizer 输出的离散符号。一个 token 可能是一个词、词片段、标点、空格模式、字节片段或特殊符号。
Vocabulary size V token id 的取值范围。它决定 embedding table 的行数，也通常影响输出 softmax 的维度。
Context length T 模型一次前向最多处理的 token 数。它不是字符数，也不是自然语言词数。
Byte fallback tokenizer 无法直接覆盖某些文本时，可退回到 byte 级表示，避免 out-of-vocabulary 崩掉。

最常用公式

问题快速估算备注
Token embedding 参数 V * d_model 每个 token id 一行向量。
LM head 参数 V * d_model 若与 embedding tied，则共享这块权重。
每步训练 token 数 batch_size * seq_len 有 gradient accumulation 或 data parallel 时再乘对应因子。
总训练 token 数 tokens_per_step * steps scaling law 讨论中最常用的数据量单位。
Perplexity exp(loss) 前提是 loss 以 nats/token 计。
Dense Transformer 训练 FLOPs 粗估 ~ 6 * params * tokens 只作预算量级判断；具体实现会受序列长度、attention 和硬件效率影响。

检查清单

  1. 特殊 token 是否明确：BOS、EOS、PAD、UNK 是否真的需要？
  2. 训练和推理是否使用同一个 tokenizer 版本？
  3. loss 是否按预测 token 平均，而不是按 batch 或 sequence 总和误读？
  4. 比较两个模型 loss 时，tokenizer 是否一致？如果不一致，是否需要 bits-per-byte 等更稳的单位？
  5. 估算训练预算时，tokens、steps、batch、context length 有没有混用？
